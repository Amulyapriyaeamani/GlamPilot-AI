# GlamPilot AI – Security Specification

## 1. Data Invariants

- **User Profiles (`/users/{userId}`)**: Users can only read and write their own profile document. The document ID must match the `request.auth.uid`. No self-assignment of high-privilege administrative flags (`isAdmin`, `isOwner`) is permitted.
- **Salons (`/salons/{salonId}`)**: Publicly read-only. Updates and creations are restricted to authorized salon administrators and platform admins.
- **Services (`/services/{serviceId}`)**: Publicly read-only. Updates are restricted to the verified salon owner or administrators.
- **Reviews (`/reviews/{reviewId}`)**: Authenticated users can write reviews provided that the `user_id` inside the review coincides with `request.auth.uid`. Reviews are publicly readable.
- **Bookings (`/bookings/{bookingId}`)**: Created by authenticated, email-verified customers. A booking can only be read or written by the purchasing customer (`user_id == request.auth.uid`) or the salon merchant who owns the salon (`salon_id`).
- **AI Consultations (`/consultations/{consultationId}`)**: Authenticated users can write. A consultation is readable only by the customer who submitted it.
- **AI Recommendations (`/recommendations/{recommendationId}`)**: Generated by the system backend, read-only for client SDKs.
- **Salon Analytics (`/analytics/{analyticsId}`)**: Restricted read-write access. Only available to the specific salon merchant who owns the salon and platform admins.
- **AI Insights (`/insights/{insightId}`)**: Read-only for authorized salon merchants; system-controlled on writes.

## 2. The "Dirty Dozen" Malicious Payloads

Here are 12 specific hostile payloads designed to compromise Identity, Integrity, or State on GlamPilot AI, and why our Fortress Rules will block them with a `PERMISSION_DENIED`.

### Payload 1: Profile Hijacking (Identity Spoofing)
- **Target Collection**: `/users/attacker-uid`
- **Attempt**: Attacker logs in as `attacker-uid` but attempts to overwrite `sindhuboddu8@gmail.com`'s profile or inject another owner's ID.
- **Payload**:
  ```json
  {
    "user_id": "victim-uid",
    "full_name": "Impostor Admin",
    "email": "victim@gmail.com",
    "phone": "9999999999",
    "created_at": "2026-06-15T12:00:00Z"
  }
  ```
- **Defense**: Blocked because the document path variable `{userId}` MUST match `request.auth.uid` and `incoming().user_id == request.auth.uid`.

### Payload 2: Admin Privilege Escalation (Self-Assigned Roles)
- **Target Collection**: `/users/attacker-uid`
- **Attempt**: Attacker registers a user profile but injects a custom high-privilege administrative property or role.
- **Payload**:
  ```json
  {
    "user_id": "attacker-uid",
    "full_name": "Malicious User",
    "email": "attacker@gmail.com",
    "role": "admin",
    "isAdmin": true
  }
  ```
- **Defense**: The strict schema validation block `isValidUser(incoming())` does not allow the fields `role` or `isAdmin` to be registered.

### Payload 3: Shadow Update / Untrusted Modification
- **Target Collection**: `/users/attacker-uid`
- **Attempt**: Attacker updates their profile but includes a bypass parameter field `"ghost_pass": true` to see if the rules allow dynamic unstructured field poisoning.
- **Payload**:
  ```json
  {
    "user_id": "attacker-uid",
    "full_name": "Attacker",
    "email": "attacker@gmail.com",
    "ghost_pass": true
  }
  ```
- **Defense**: Blocked by strict strict-key rules (`affectedKeys().hasOnly(...)`) and exact size verification.

### Payload 4: Booking Sabotage (Cross-User Write)
- **Target Collection**: `/bookings/victim-booking-123`
- **Attempt**: Auth user `attacker-uid` attempts to cancel or modify a booking belonging to `victim-uid`.
- **Payload**:
  ```json
  {
    "booking_id": "victim-booking-123",
    "user_id": "victim-uid",
    "salon_id": "salon-1",
    "booking_status": "Cancelled"
  }
  ```
- **Defense**: Denied because `existing().user_id == request.auth.uid` is required for non-merchant updates, and the attacker does not own this document.

### Payload 5: Spoofed Review / Fake Reputation Posting
- **Target Collection**: `/reviews/hostile-review-99`
- **Attempt**: Attacker attempts to post a 1-star review for a competitor, impersonating a well-known premium client uid.
- **Payload**:
  ```json
  {
    "review_id": "hostile-review-99",
    "salon_id": "salon-2",
    "user_id": "trusted-client-uid",
    "rating": 1,
    "review_text": "Horrible service and unsafe layout!",
    "review_date": "2026-06-15"
  }
  ```
- **Defense**: Denied because our `isValidReview` validation function mandates `incoming().user_id == request.auth.uid`.

### Payload 6: Price Tampering (Denial of Wallet)
- **Target Collection**: `/bookings/attacker-booking-456`
- **Attempt**: Attacker creates a new appointment for a luxury ₹9,500 Bridal package but sets the `total_amount` to ₹1.
- **Payload**:
  ```json
  {
    "booking_id": "attacker-booking-456",
    "user_id": "attacker-uid",
    "salon_id": "salon-1",
    "booking_date": "2026-07-20",
    "booking_time": "3:00 PM",
    "total_amount": 1,
    "booking_status": "Confirmed"
  }
  ```
- **Defense**: Denied through transaction validation matching and server-side verification checks during booking synchronization.

### Payload 7: Terminal State Bypass / Re-activation
- **Target Collection**: `/bookings/booking-789`
- **Attempt**: A booking is marked as `"Completed"`. The user attempts to demote its status back to `"Pending"` or change details of a completed work.
- **Payload**:
  ```json
  {
    "booking_id": "booking-789",
    "user_id": "attacker-uid",
    "salon_id": "salon-1",
    "booking_date": "2026-07-20",
    "booking_time": "3:00 PM",
    "total_amount": 9500,
    "booking_status": "Pending"
  }
  ```
- **Defense**: Blocked because once `existing().booking_status == "Completed"` is terminal, further client updates are frozen.

### Payload 8: Service Pricing Alteration
- **Target Collection**: `/services/s-1-1`
- **Attempt**: Attacker tries to modify the public catalog price of a luxury service.
- **Payload**:
  ```json
  {
    "service_id": "s-1-1",
    "salon_id": "salon-1",
    "service_name": "Bridal Makeup",
    "price": 100
  }
  ```
- **Defense**: Blocked because non-admins or non-verified merchants are prohibited from updating `/services/`.

### Payload 9: Malicious Large Payload Attack (Resource Poisoning)
- **Target Collection**: `/consultations/overflow-id`
- **Attempt**: Attacker attempts to flood the Firestore database by injecting a massive 10MB string as the `notes` value.
- **Payload**:
  ```json
  {
    "consultation_id": "overflow-id",
    "occasion": "Wedding",
    "budget": 5000,
    "location": "Jubilee Hills",
    "notes": "A".repeat(1000000),
    "created_at": "2026-06-15T12:00:00Z"
  }
  ```
- **Defense**: Blocked by maximum string size enforcements: `incoming().notes.size() <= 2000`.

### Payload 10: ID Poisoning (Path Variable Hijacking)
- **Target Collection**: `/salons/invalid%20character$@*!()`
- **Attempt**: Attacker calls a document creation with an illegal/malicious special characters string as the ID to create corrupted cache or index files on Firestore.
- **Payload**:
  ```json
  {
    "salon_id": "invalid%20character$@*!()",
    "salon_name": "Vandal Salon",
    "area": "Jubilee Hills"
  }
  ```
- **Defense**: Blocked because all write operations validate `{id}` using `isValidId()`, which enforces `id.matches('^[a-zA-Z0-9_\\-]+$')`.

### Payload 11: Spoofed Analytics Insertion
- **Target Collection**: `/analytics/an-1`
- **Attempt**: Competitor tries to rewrite or delete a salon's metrics to damage their dashboard standing.
- **Payload**:
  ```json
  {
    "analytics_id": "an-1",
    "salon_id": "salon-1",
    "revenue": 0
  }
  ```
- **Defense**: Only authorized platform creators or associated salon owners are allowed read/write permissions on `/analytics/`.

### Payload 12: Cheat-Code Customer Review Rating
- **Target Collection**: `/reviews/rvw-404`
- **Attempt**: Hostile reviewer tries to post an extreme out-of-bounds rating number (e.g. `rating: 100` or `rating: -5`).
- **Payload**:
  ```json
  {
    "review_id": "rvw-404",
    "salon_id": "salon-1",
    "rating": 100,
    "review_text": "This rating will break their dashboard!",
    "review_date": "2026-06-15"
  }
  ```
- **Defense**: Denied because our review validation strictly enforces `incoming().rating >= 1 && incoming().rating <= 5`.
